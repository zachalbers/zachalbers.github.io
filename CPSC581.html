Project 0: Homer Simpson


Initial Sketches:

Project 0 started with every group member sketching 10 different ideas, from which we would have a large selection to move forward with. I started by interviewing some of my classmates to generate some initial sketches and also experimented with some celebrity ideas.




Further Sketching and Iteration:

Once each team member had generated lots of ideas, we came together to choose an idea to go forward with. Some interesting sketches had been done on Homer Simpson, so we decided to pursue that character and sketch some ideas further off of it.







Implementation:


After everyone created additional sketches, we met together and brainstormed additional ideas and came to a consensus on what we were going to create. Project 1: Creative Unlock Apps


Initial Sketches:

Just as in project 0, for this project we started with every group member sketching 10 different ideas for creative ways to unlock your phone. We had to think of ideas that would work by touching the phone and ones that would work using the sensors on the device. My initial designs can be seen here.












Further Sketching and Iteration:

After we had generated a lot of different ideas, we met as a group early on to discuss the ideas we wanted to move forward with. We decided to move forward with a compass-based idea for the sensor project and music note tapping-based idea for the second project. Implementation:


After receiving some feedback from the class and brainstorming different iterations, we decided to go with an palette and spinning brush idea, and a falling music note idea. The final implementation of these can be seen below. To get the final iterations, there were a lot of design decisions that had to be made. For the music app, we decided to have unique symbols and colors with little similarities so that users could easily distinguish them. For the artboard app, we wanted it to be clear which symbols were being chosen which is why we decided to both highlight the color and point to it with the brush in the centre.

For implementing this project, I largely worked on programming the brush movement and animation, and the working with the iPhone sensors to move the brush accordingly. I also programmed the logic behind the color choosing and password system.

Project 2: Washing Machine Notifier


Initial Sketches:

For this project, each member of the group started out by sketching 10 different ideas. This project required us to solve an 'everyday' problem, using an arduino and some type of sensor. For my ideas, I found a lot of the ones I thought up of involved some kind of app, which turned out being more complicated to implement. However I found that the ideas involving an app were the ones I was most likely to use. My initial designs can be seen here. (Still working on getting them to rotate!)
















Further Sketching and Iteration:

Once we all generated lots of different ideas, we met in class to discuss our favorites, and then presented them to class. After receiving class feedback, we decided to go sketch some more iterations for the backpack-based sensor/notification idea, and the washing machine sensor/notification idea (which is what we ended up going with). Here are some of the sketches from this stage:



















Implementation:


After reviewing our refined sketches and discussing some more, we decided to go forward with the washing machine idea. The idea for this was to have a sensor on the washing machine that could notify the user once it had finished. Many people find it annoying to leave their clothes in the washer or dryer, which can leave them with mildew or wrinkles. This product solves that problem, but notifiying a person when the cycle of the machine is done. Ideally, there would be a sensor on each of the washer and dryer, as well as a receiver for each household memeber, but due to limited time and materials we only made one sensor and receiver. We wanted it to be discrete for the user, so we encased both the sensor on the washing machine and the receiver held by the user in black boxes to hide the wires. Inside the first box, which is placed on the washing machine with magnets, contains the vibration sensor and has the on/off switch, which once activated, will signal the user once the washing machine cycle has ended. The box that is carried on the user has a light and a buzzer that lights up and plays a Mario-themed song to let the user know when the washing machine is finished.

For implementing this project, we largely worked in the lab to brainstorm, design, program, and debug, since there was only one arduino board that could be worked on. However, this proved to be productive and we were able to finish the majority of the project in the lab.
