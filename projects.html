UniMap + Map Builder

This page is still under Development


Summary

A map application that displays the quickest route to travel between two locations, input by a user, in a graphical interface of the map, as well as the estimated time taken to get there. UniMap was originally designed to give a map of the University of Calgary(hence the name), but has been modified to take in a text configuration file of the desired place a user would want to map. The user is responsible for providing their own text configuration files with the exception of the defaults provided.

UniMap will display a map(based on the text file selected by the user) along with a path from the user's current selected building to their destination building. If the user has decided to add a legend then the time required by either walking, running, skateboarding, or biking will also be displayed along with icons.







My Roles in the Project

  • Brainstormed and helped come up with initial ideas and the implementation plan for the project
  • Designed and programmed the logic for the map window, including the map legend, buildings, and highlighted path
  • Designed and programmed map data structures & pathfinding logic

Tools/Platforms Used

  • Atom
  • Github


What I Learned

This project was the first in which I had the opportunity to spend a lot of time in the Java language and so was able to really learn a lot about application structure, alogrithm design, and data manipulation.

Project 2: Washing Machine Notifier


Project Summary:


A project built using Arduino that allows a user to be notified when their laundry is done. The purpose behind this project was to create a physical prototype with Arduino to solve an everyday problem.



Video Demonstration:



Initial Sketches:

For this project, each member of the group started out by sketching 10 different ideas. This project required us to solve an 'everyday' problem, using an arduino and some type of sensor. For my ideas, I found a lot of the ones I thought up of involved some kind of app, which turned out being more complicated to implement. However I found that the ideas involving an app were the ones I was most likely to use. My initial designs can be seen here. (Still working on getting them to rotate!)
















Further Sketching and Iteration:

Once we all generated lots of different ideas, we met in class to discuss our favorites, and then presented them to class. After receiving class feedback, we decided to go sketch some more iterations for the backpack-based sensor/notification idea, and the washing machine sensor/notification idea (which is what we ended up going with). Here are some of the sketches from this stage:



















Design & Implementation:


After reviewing our refined sketches and discussing some more, we decided to go forward with the washing machine idea. The idea for this was to have a sensor on the washing machine that could notify the user once it had finished. Many people find it annoying to leave their clothes in the washer or dryer, which can leave them with mildew or wrinkles. This product solves that problem, but notifiying a person when the cycle of the machine is done. Ideally, there would be a sensor on each of the washer and dryer, as well as a receiver for each household memeber, but due to limited time and materials we only made one sensor and receiver. We wanted it to be discrete for the user, so we encased both the sensor on the washing machine and the receiver held by the user in black boxes to hide the wires. Inside the first box, which is placed on the washing machine with magnets, contains the vibration sensor and has the on/off switch, which once activated, will signal the user once the washing machine cycle has ended. The box that is carried on the user has a light and a buzzer that lights up and plays a Mario-themed song to let the user know when the washing machine is finished.

For the design of this project we wanted something that was discrete, and only became noticable when the device was notifying the user of the washing machine being done. There were other ideas that we discussed, such as having a set of buttons on the washing machine, corresponding to each person in the house and another receiver, that would be notified when the wachine wachine was finished. However, this would have been much larger then the design we ended up going with and would require and a larger number of recievers. For the reciever itself, there were ideas to have it attached to the wall of the room, but we decided that a device that could be carried in a person's pocket would be more practical.

For implementing this project, we largely worked in the lab to brainstorm, design, program, and debug, since there was only one arduino board that could be worked on. However, this proved to be productive and we were able to finish the majority of the project in the lab.

Project 3: Life Detector


Project Summary:


A project built using Arduino that displays the heartbeat pattern of the user, and changes color and speed to reflect the person's heartrate. The instructions for this project was to create a fashion tech garment using Arduino that has at least one input and output sensor.



Video Demonstration:



Initial Sketches:

To come up with a large selection of ideas to choose from we each sketched 10 different ideas. For my initial sketches I thought of primarily LEDs as the output, as this would be the most feasible to implement given the resources available to us. Some of my ideas involved inputs such as geo-location, touch, heartbeat, light, or a specific motion. I liked the idea of having visual feedback with the lights in response to some input, so a lot of my ideas contained some form of lights turning on in sync with the users input.











Further Sketching and Iteration:

We met in class after having sketched out different ideas, to compare and receive feedback from to choose good ideas to further iterate with. We presented our favorites to the class and based on feedback we received decided to sketch some more ideas for a Life Detector idea and a LED hair extension idea. Here are the sketches for these ideas:



















Design & Implementation:


After reviewing the refined sketches, we discussed how each might be implemented and decided that we could best implement the heartbeat sensor idea both aesthetically and functionally. Once we decided on that idea to go forward with, we discussed some possibilites for the design.

We wanted the the heartbeat to be visible when on, but didn't want it stand out when it was off. To accomplish this we decided to stitch (and later glue) the lights to an undershirt, and then have another shirt to go on top (seen below). This way we could shape the lights into a heart on the undershirt without worrying about hiding wires. Some other things we tried to make the lights look better included black felt and tape, but we eventually used glue to hold the wires down, which allowed the light defuse more evenly and more resemble the shape of a heart through the top shirt.

For the code design, we were able to utilize some PulseSensor examples and online tutorials to figure out how to monitor both the heartbeat and heartrate. We wanted the heartrate level to correspond with different colors in the LED strip to visibly indicate that the heartrate is low (below 70), medium (70-100), or high (over 100), so we programmed the LED strip to change colors (once a BPM could be calculated) according to the wearers heartrate.

The idea behind this project was to have a visible representation of the wearers heartbeat. To accomplish this we programmed the LEDs to 'pulse' in succession, to show the speed of the heartbeat but still maintain the shape of a heart. A heartbeat also reveals information about the wearers state, so to represent this we had a darker blue color which suggests a calmer aura. As the the hearbeat increases the color changes to Magenta and the a bright red which suggests more actvity. We liked the clean look of a white t-shirt, which would also allow the LED lights to brightly enough, while being thick enough to hide them when off.
